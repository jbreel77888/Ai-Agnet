## 0. الملخص التنفيذي

المشروع لديه **أساس معماري ممتاز على الورق** — Clean Architecture، DI Container، EventBus، Drizzle + PostgreSQL + pgvector، 9 أنواع وكلاء موثقة، 17 أداة موثقة في `TOOLS.md`.

**الواقع في الكود**: منصة MVP عاملة بنسبة ~35% من رؤية Manus.

- Chat SSE يعمل، Thinking Panel حقيقي، ToolCards تعرض.
- 9 وكلاء **معطّلين في DB — يعمل وكيل واحد فقط هو Planner**.
- 7 أدوات فقط مسجلة فعليًا، 3 منها stub ضعيف.
- لا يوجد CodeAct، لا VM sandbox حقيقي، لا Computer Use، لا filesystem آمن، لا RAG end-to-end، لا browser VNC.
- لا MCP servers متصلة.
- لا Human-in-the-Loop، لا Tool Permissions، لا Audit حقيقي.

للوصول لمستوى Manus AI / Kimi / Z.ai تحتاج **3 طبقات ناقصة بالكامل**: Computer/Sandbox Layer، Tool Ecosystem Layer، Multi-Agent Orchestration Layer.

## 2. تحليل الوكلاء Agents

**الموثق في AGENTS.md:**
Planner, Research, Reasoning, Coding, Execution, Tool, Memory, Reflection, Summarizer

**الفعلي في الكود:**

`src/agents/registry/index.ts` — يسجل الوكلاء من DB
`src/db/seed/agents.ts` — يزرع 9 وكلاء، لكن في `Chat page` التعليق واضح:
> "Single universal agent (the planner). The platform auto-selects tools"

`src/agents/router/index.ts` — يوجد router بسيط keyword-based:
```ts
trip|itinerary|agenda → planner
```
لا scoring حقيقي، لا `canHandle(input): number 0-1`.

**النواقص الحرجة:**
1. لا Handoff Payload حقيقي — لا تمرير contextSnapshot
2. لا Sub-Agents spawning — الكود موجود كـ interface فقط
3. لا Agent Events كاملة: `thinking`, `tool_call`, `tool_result`, `completed` فقط — ناقص `handoff_request`, `subagent_spawned`
4. لا Recursive Tasks، لا maxRecursionDepth guard
5. لا Agent Metrics / HealthCheck
6. لا Shared Context مع variables / artifacts / budget
7. الـ Orchestrator يرسل مباشرة لـ ProviderManager.chatStream، لا يمر على Agent Engine

**مقارنة Manus:**
Manus يستخدم **multi-agent CodeAct**: وكيل Planner يولد plan، ثم sub-agents متوازية (Research, Coding, Browser)، مع Reflection loop. كل sub-agent لديه VM خاص. لا يوجد هذا هنا.

**التوصية:**
- فعل الـ 9 وكلاء في DB فورًا
- نفذ `AgentEngine.execute()` الحقيقي بـ event streaming
- نفذ Router بـ LLM-based classification + tool detection
- أضف Handoff Manager + SubAgentPool
- Budget guard: maxTotalSteps=100، maxTotalCost=$10

## 4. Computer / Sandbox — مفقود بالكامل

Manus AI قوته: **"يعمل على كمبيوتر مستقل"** — VM كامل مع filesystem، browser، terminal، code editor.

مشروعك: لا يوجد.

- Tensorlake اختياري فقط
- لا E2B / Daytona / CodeSandbox
- لا session-scoped workspace حقيقي (يوجد `/api/sessions/[id]/files` لكنه DB فقط، لا filesystem)
- لا terminal streaming
- لا browser VNC live view

**الحل:**
ادمج **E2B Code Interpreter SDK**:
```ts
npm i @e2b/code-interpreter
```
كل session = sandbox جديد:
- `/workspace` persistent
- Python + Node + bash
- pip/npm install
- matplotlib charts → artifacts
- files API يربط مع الـ sandbox

هذا يحول مشروعك من "chatbot مع tools" إلى "Manus-like agent computer".

## 13. مشاكل حرجة يجب إصلاحها فورًا

1. **Prisma vs Drizzle تضارب** — احذف `prisma/` بالكامل
2. **calculator tool XSS** — `Function('Math', ...)` خطر، استخدم `mathjs`

8. **JWT_SECRET في .env؟** — تأكد أنه مولد بـ `openssl rand -hex 32`
9. **Playwright في dependencies الرئيسية** — 200MB، انقله لـ optionalDeps
10. **`z-ai-web-dev-sdk` في package.json** — ما هذا؟ غير موثق، احذفه
