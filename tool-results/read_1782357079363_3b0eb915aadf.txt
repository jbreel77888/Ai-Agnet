    32→## 1. ملخص تنفيذي
    33→
    34→منصة Ai Norx مبنية على LibreChat مع تخصيصات عربية لإجبار جميع المستخدمين العاديين (USER) على استخدام وكيل مركزي واحد باسم `agent_primary`. التحكم بهذا الوكيل يتم حالياً عبر:
    35→
    36→- **سطر ثابت في الكود** (`PRIMARY_AGENT_ID = 'agent_primary'`) متكرر في 4 ملفات على الأقل (Frontend + Backend).
    37→- **سكربت CLI يدوي** (`config/setup-primary-agent.js`) يكتب تعليمات عربية hardcoded ويعيد كتابتها عند كل تشغيل، مما يلغي أي تعديلات يجرها الـ Admin من الواجهة.
    38→- **لا يوجد حقل `isDefault`** في مخطط `Agent` ولا يوجد API endpoint لتعيين وكيل افتراضي.
    39→- **لا توجد قاعدة بيانات للمزودين/النماذج** — كل شيء في `librechat.yaml`.
    40→
    41→**النتيجة الحالية:**
    42→- زر الوضع العلوي للـ Admin = `<ModelSelector>` كامل (specs + endpoints + models + agents) — مخالف للمطلوب.
    43→- زر الوضع العلوي للـ USER = نص ثابت "Ai Norx" غير قابل للنقر — مخالف للمطلوب (المطلوب: خياران Agent Mode / Chat لكلا الطرفين).
    44→- واجهة Marketplace للـ Admin = browse-only بلا زر "إضافة"، والإضافة الحقيقية مدفونة في Side Panel منفصل.
    45→- لا يوجد مكان لتعيين وكيل افتراضي من الواجهة.
    46→- لا توجد واجهة لإضافة مزودين/نماذج عبر الـ UI (YAML فقط).
    47→- مشكلة "تحديث الـ Agent عند فتح محادثة" لم تُثبت كـ per-conversation، لكن يوجد خطر حقيقي من `setup-primary-agent.js` الذي يُعاد تشغيله عند كل deploy ويمحى تعديلات الـ Admin.
    48→
    49→التقرير التالي يقدم خطة إصلاح كاملة على 5 مراحل مع مخططات schemas جديدة و endpoints جديدة وتغييرات ملفات دقيقة.

   266→### 3.3 التدفق البرمجي عند بدء محادثة Agent جديدة

   480→### 6.1 ما قاله المستخدم بالضبط

   484→### 6.2 نتيجة الفحص الفعلي
   485→
   486→**بعد تتبع كامل التدفق:** لا يوجد كود في الباك إند يكتب على collection `agents` أثناء إنشاء محادثة جديدة. الكتابات الوحيدة على `agents` هي:

   488→2. تحديث avatar URL فقط (`skipVersioning: true`).
   489→3. سكربت CLI `setup-primary-agent.js`.

   491→### 6.3 التفسيرات الممكنة
   492→
   493→| التفسير | الاحتمالية | الدليل |
   494→|---|---|---|
   495→| **(أ) السكربت `setup-primary-agent.js` يُعاد تشغيله عند كل deploy** | عالية جداً | السكربت upsert كامل لـ `agent_primary` مع التعليمات العربية hardcoded (الأسطر 46–109) — كل تعديلات الـ Admin على هذا الـ Agent تُمحى. المستخدم قد يلاحظ هذا بعد كل deploy ويعتقد أنه يحدث "عند فتح محادثة" |
